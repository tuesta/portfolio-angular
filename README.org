* Proyecto Angular: Portfolio Profesional con Arquitectura Moderna

** 1. Objetivo General
Este proyecto transforma un portfolio profesional en una Single-Page Application (SPA) dinámica utilizando Angular. El objetivo es construir una solución robusta, escalable y mantenible, aplicando principios avanzados de componentización, enrutamiento, gestión de estado y buenas prácticas de desarrollo con TypeScript.

** 3. Estructura del Proyecto
La arquitectura del proyecto está diseñada para ser modular y escalable, separando claramente las responsabilidades.
- =src/app/components=: Contiene componentes reutilizables y de presentación (Dumb Components).
  - =nav-filters=: Muestra las etiquetas de filtrado. Es un componente reutilizable que recibe los datos a mostrar.
  - =theme-toggle=: Botón para cambiar entre el tema claro y oscuro.
  - =language-toggle=: Botón para cambiar entre español e inglés.
  - =post-content=: Renderiza el contenido de un artículo en español.
  - =post-content-en=: Renderiza el contenido de un artículo en inglés.
- =src/app/core=: Contiene la lógica de negocio, servicios y datos transversales a toda la aplicación.
  - =services/theme.ts=: Servicio para la gestión del tema visual de la aplicación.
  - =services/language.ts=: Servicio para la gestión del idioma de la aplicación.
  - =constants/posts.data.ts=: "Base de datos" estática de los artículos del blog.
- =src/app/pages=: Contiene los componentes inteligentes (Smart Components) que actúan como páginas principales.
  - =index-tag=: Página que muestra la lista de artículos, ya sea completa o filtrada por una etiqueta.
  - =post=: Página que muestra el contenido de un artículo individual.

** 4. Funcionalidades Clave

*** 4.1. Sistema de Temas (Claro/Oscuro)
Se ha implementado un sistema de temas para mejorar la experiencia de usuario, permitiéndole elegir entre un modo claro y uno oscuro.
 - *Servicio de Tema (=ThemeService=)*: Ubicado en =src/app/core/services/theme.ts=, este servicio gestiona toda la lógica del tema.
   1. *Detección Automática*: Al iniciar, el servicio detecta la preferencia de tema del sistema operativo del usuario a través de =window.matchMedia('(prefers-color-scheme: dark)')=.
   2. *Persistencia*: La elección del usuario se guarda en =localStorage=, asegurando que su preferencia se mantenga en futuras visitas.
   3. *Actualización Dinámica*: El servicio utiliza una =signal= de Angular para mantener el estado actual del tema. Al cambiarlo, se actualiza un atributo =data-theme= en el elemento =<html>=, lo que permite que los estilos (=styles.scss=) apliquen los colores correspondientes de forma reactiva.

*** 4.2. Filtrado de Contenido por Etiquetas
La aplicación permite una navegación fluida y un descubrimiento de contenido intuitivo a través de un sistema de filtrado basado en etiquetas, que está profundamente integrado con el sistema de enrutamiento de Angular.
- *Enrutamiento Dinámico*: El archivo =src/app/app.routes.ts= define las rutas:
  - =/=: Muestra todos los artículos.
  - =/tag/:slug=: Muestra los artículos que coinciden con la etiqueta (slug).
- *Componente de Navegación (=NavFilters=)*: El componente =src/app/components/nav-filters/nav-filters.ts= es puramente presentacional. Recibe la lista de todas las etiquetas disponibles (=tags=) y las renderiza como enlaces de navegación.
  - Utiliza =RouterLink= y =RouterLinkActive= para generar los enlaces y resaltar la etiqueta activa, delegando completamente el estado de la aplicación al enrutador de Angular.
- *Página de Índice (=IndexTag=)*: Esta página inteligente obtiene la etiqueta de la URL (o ninguna, si es la página principal) y filtra los datos de =POSTS_DATA= para mostrar solo los artículos correspondientes.

*** 4.3. Soporte Multi-idioma
La aplicación ha sido actualizada para ofrecer una experiencia multi-idioma, permitiendo a los usuarios cambiar entre español e inglés.
- *Servicio de Idioma (`LanguageService`)*: Ubicado en `src/app/core/services/language.ts`, este servicio gestiona el idioma de la aplicación.
  1. *Detección de Idioma del Navegador*: Al iniciar la aplicación, el servicio detecta el idioma preferido del navegador del usuario.
  2. *Persistencia*: La selección de idioma del usuario se almacena en `localStorage` para mantener la preferencia en futuras visitas.
  3. *Cambio Dinámico*: A través de un componente de palanca (`LanguageToggleComponent`), el usuario puede cambiar el idioma en cualquier momento. El estado del idioma se gestiona con una `signal` de Angular, que permite una actualización reactiva de la interfaz.
- *Contenido Traducido*: El contenido de los artículos se ha duplicado y traducido.
  - `post-content`: Muestra el contenido en español.
  - `post-content-en`: Muestra el contenido en inglés.
  - La página `post` renderiza dinámicamente uno de los dos componentes anteriores según el idioma seleccionado.

** 5. Justificación de Decisiones de Diseño
El diseño actual se desvía de una estructura de portfolio tradicional ("Home", "Portfolio", "Contacto") para adoptar un enfoque más simple y centrado en el contenido, similar a un blog técnico.
 - *Simplicidad y Claridad*: Se eliminaron secciones que no aportaban valor al objetivo principal (mostrar proyectos y artículos) para evitar la sobrecarga de información y reducir la complejidad. El "Portfolio" y el "Home" se fusionan en la página principal, que actúa como un índice de todo el contenido.
 - *Enfoque en el Contenido*: La arquitectura está optimizada para la presentación de artículos. Funcionalidades como el filtrado por etiquetas y el cambio de tema están diseñadas para mejorar la experiencia de lectura y descubrimiento.
 - *Sin Formulario de Contacto*: Se ha prescindido de un formulario de contacto para simplificar el mantenimiento y porque el contacto puede establecerse a través de los enlaces en el pie de página (redes sociales, email), que son más directos y fáciles de mantener.
